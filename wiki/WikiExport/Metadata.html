Metadata

Associated with both Recordings and Signals. Held as RDF (http://www.w3.org/TR/rdf-concepts/)

Each recording has its own named graph, holding the metadata of the recording and its associated signals.

Implementation

The Jena framework (http://jena.sourceforge.net/index.html) is used to hold and query RDF. In full BioSignalML implementation (ie. one with a metadata repository), a persistent RDF store is provided via Jena's TDB layer; it could equally be provided by Jena's SDB and a SQL database, or by products such as Virtuoso RDF (http://virtuoso.openlinksw.com/dataspace/dav/wiki/Main/VOSRDF) which have a Jena interface. For a partial BioSignalML implementation there is no persistence layer (outside of what a Recording's data file may contain), and RDF is held in an in-memory model.
