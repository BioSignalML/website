HDF5-based recording format

A Recording can be stored in HDF5 format.
  • One recording in a HDF5 file.
  • Recordings can be opened or created.

    Signals are held as HDF5 groups containing signal segment datasets. Operations on signals are:
    • create -- needs to know format/type of point's values.
    • open -- will return/set format/type of values.
    • append -- data points (as signal segments; if timed segment then wrt signal start; if uniform segment then have sample-rate; if non-uniform then refer to sample timing (which is held in a separate group/dataset)).
    • position -- by time (and by point offset ??).
    • read -- return a sequence of timed segments.
Core metadata associated with a signal is held within the signal's HDF5 group.
