Signal Naming

  • A Signal is considered to be a single data stream from some source (transducer, simulation, etc).
  • A Recording is a container for a set of Signals all of which have been recorded over the same time period and pertain to the one subject (or simulation run).
  • Each Signal is part of some Recording.
  • Each Signal has an identifier that is unique within the Recording that the Signal is part of.
  • Each Recording has an URI (http://tools.ietf.org/html/rfc3986) that has no query or fragment component. A recording URI should not end in a slash ("/").
  • A Signal's URI is formed by adding its identifier as the query component of a Recording's URI.
  • The fragment identifier of a Signal's or Recording's URI is used to denote a range of times for which to retrieve information.
  • A time range consists of a sequence of time intervals and instants seperated by commas (",")
  • A time instant is given as the duration from the start of the Recording (that the Signal is in) to the time point.
  • A time interval is either in the form [ start ":" length ] or [ start "-" end ] where 'start' and 'end' are the duration from the start of the Recording and 'length' is the duration of the interval.
  • Durations are formatted as an XML Schema Datatype duration (http://www.w3.org/TR/xmlschema-2/#duration (ISO 8601 extended format)).

Examples:

  • A recording: http://my.domain.org/path/to/recording
  • The ecg signal in the recording: http://my.domain.org/path/to/recording?ecg
  • Signal 3 in swa49.edf: file:///Users/dave/Desktop/SWA/swa49/swa49.edf?3
  • Signal 3 in swa49.edf for the 23.14 second long interval starting at 5 minutes 3.57 seconds: file:///Users/dave/Desktop/SWA/swa49/swa49.edf?3#PT5M3.57S-PT23.14
  • A relative reference to the recording and specifying a time interval: file://./swa49/swa49.edf#PT5M3.57S-PT23.14
