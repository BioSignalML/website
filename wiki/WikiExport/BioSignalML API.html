BioSignalML API

General aim is simplicity of use, certainly to access the most common functions.
  • User has a signal file (recording) and wants to get signal data from it.
  • User does some query or search to get the signal file(s) and signal id(s) and time intervals of interest; then wants signal data for visualisation or prococesing.
  • Having processed signal data resulting data needs to be stored into a new recording (signal file).



Use cases:
  1. Only import from a single package for simple usage.
  2. Work with a repository:
      val repo = new typeRepository(location, connection parameters)  // jenaRepository etc
                                                                  // sparqlRepository ??
  repo.query(sparql)
  val rec = repo.getRecording(record_url)
  for (r <- repo.getRecordings()) { }
  val sig = rec.getSignal(signal_id)
  val sig = repo.getSignal(record_url#signal_id)
  for (s <- rec.getSignals()) { }
  sig.units()
  sig.getData(interval)   // etc
  3. Work directly with a signal file:
      val rec = new typeRecording(location)                           // EDFRecording
  val sig = rec.getSignal(signal_id)
  for (s <- rec.getSignals()) { }
  sig.units()
  sig.getData(interval)   // etc
  4. Attach a signal file to a repository and work with the repository:
      val repo = new typeRepository(location, connection parameters)  // jenaRepository etc
  val rec = new typeRecording(location)                           // EDFRecording
  repo.attach(rec)
  repo.query(sparql)
  // etc

    Do we have functions such as:
