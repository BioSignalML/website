Repositories

Difference between model repositories, model development and saving workspaces, and a signal repository -- model development is akin to software development or prototype construction, trying out ideas and concepts; a signal repository is more a library of recordings -- the contents can be used as input for modelling, or for analysis.

Signal stores

The container object used to hold and access signals/recordings will be known as a Store instead of as a Repository. A Repsitory could be constructed from one or more Stores by providing additional metadata and functions (for curation, collaboration and provenance). The best way to do so could be to use the PMR2 repository as a wrapper around a Store.
