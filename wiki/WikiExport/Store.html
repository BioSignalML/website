Store

A Store is the container used to hold and access signals and recordings. A recording may either be held within a store (by being imported), or it may be linked or associated with a store whereby the recording's information (signals/data/metadata) is able to be accessed via the store, without an actual import of data. In this case the link is terminated at the end of a session.

Directory structure:
  store_directory/
    recordings/
      # sub-directory holding imported recordings (in HDF5 format?)
    metadata/
      # sub-directory with metadata held as a Jena repository
Access via API:
  val store = new Store("path_to_store_directory")
  ...
  store.close()
Different views of a Store and its contents:
  • Signal/recording files
  • Metadata
  • Streaming
  • Web services (browsing, data provider...)
  • Import/Export
