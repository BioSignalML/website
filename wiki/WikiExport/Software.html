Software

Built around core concepts (ie. core package), with a set of (quasi-)orthogonal modules for accessing signal files, metadata, streaming data and annotation (editing).

Data/metadata could be held in a store and referenced by a signal URL, or in an external file, or in stream. In these last two cases, the first step in using the signal/recording is to attach (bind?) it to the local repository.

Note: This binding is not the same as importing the signal/recording into the store -- binding is the process of adding the signal/recording into the store's namespace and making data/metadata available via the top-level API functions.
